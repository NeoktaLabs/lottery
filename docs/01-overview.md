# Overview

Ppopgi (뽑기) is designed around one core idea:

> If a user can read the chain, they should fully understand what happens.

Each raffle is a standalone smart contract with immutable rules.
Funds never leave the contract except through predefined, verifiable paths.

### Core properties
- Verifiable randomness (Pyth Entropy)
- Pull-based withdrawals
- Permissionless finalization
- No upgradeable game logic
- No admin access to user funds

Anyone can interact with the contracts directly, even without the frontend.
